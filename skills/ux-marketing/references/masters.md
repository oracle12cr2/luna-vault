# UX/Marketing 대가 기준 상세

## 1. Steve Krug "Don't Make Me Think"

웹 사용성의 바이블. 핵심: 사용자가 **생각하지 않고** 쓸 수 있어야 한다.

### 핵심 원칙

#### 사용성 제1법칙: 생각하게 만들지 마라
- 클릭할 수 있는 건 클릭할 수 있게 보여야 함
- 사용자가 "이게 버튼인가?" 고민하면 실패
- 명확한 시각적 단서: 밑줄, 색상 변화, 커서 변경

#### 스캔하게 만들어라 (읽지 않는다)
- 사용자는 읽지 않고 **스캔**한다
- 시각적 계층: 제목 > 소제목 > 본문 > 메타
- **핵심을 먼저**: 역피라미드 구조
- 키워드 볼드 처리, 불릿 리스트 활용
- 의미 없는 텍스트(Happy Talk) 제거

#### 3클릭 규칙
- 사용자가 원하는 곳에 **3클릭 이내** 도달
- 실패하면 이탈 가능성 급증
- 단, 각 클릭이 **쉽고 확신 있으면** 더 많은 클릭도 OK
- 핵심: 클릭 수보다 **클릭할 때의 확신**

#### 네비게이션 설계
- **글로벌 네비게이션**: 항상 같은 위치, 같은 형태
- **현재 위치 표시**: "You Are Here" — 하이라이트, 브레드크럼
- **검색 필수**: 어디서든 접근 가능한 검색창
- **홈으로 돌아가기**: 로고 클릭 = 홈 (관례)

#### 트렁크 테스트
아무 페이지에 "낙하산 투하"되었을 때 5초 내에 답할 수 있어야:
1. 이 사이트가 뭐하는 곳인지?
2. 지금 어떤 페이지에 있는지?
3. 주요 섹션(메뉴)이 뭔지?
4. 이 레벨에서 선택지가 뭔지?
5. 사이트 전체에서 내 위치가 어딘지?
6. 어떻게 검색하는지?

### 블로그 적용 체크리스트
- [ ] 모든 인터랙티브 요소가 명확히 클릭 가능해 보임
- [ ] 글 목록이 스캔 가능 (제목 > 카테고리 > 날짜 > 발췌)
- [ ] 모든 글에 3클릭 이내 도달
- [ ] 글로벌 네비게이션 일관성
- [ ] 현재 위치 표시 (카테고리 하이라이트, 브레드크럼)
- [ ] 검색 상시 접근 가능
- [ ] 트렁크 테스트 통과

## 2. Jakob Nielsen 10 Heuristics

사용성 평가의 골든 스탠다드. 1994년 정립, 30년간 검증된 기준.

### 10가지 휴리스틱

#### 1. 시스템 상태의 가시성 (Visibility of System Status)
- 현재 상태를 항상 사용자에게 피드백
- 블로그: 로딩 상태, 현재 카테고리, 페이지 번호
- 예: 스켈레톤 UI, 프로그레스 바, 활성 메뉴 하이라이트

#### 2. 시스템과 실세계의 일치 (Match Between System and Real World)
- 사용자의 언어와 관습을 따름
- 블로그: "게시물" > "포스트", "분류" > "카테고리"
- 아이콘은 보편적으로 이해되는 것만 사용

#### 3. 사용자 제어와 자유 (User Control and Freedom)
- 실수에서 쉽게 복구
- 블로그: 뒤로가기 정상 작동, 검색 초기화, 필터 리셋
- SPA에서 히스토리 관리 필수

#### 4. 일관성과 표준 (Consistency and Standards)
- 같은 말, 같은 행동, 같은 위치
- 블로그: 메뉴 위치 고정, 링크 스타일 통일, 버튼 스타일 통일
- 외부 일관성: 웹 관례 따르기 (로고=홈, 밑줄=링크)

#### 5. 에러 방지 (Error Prevention)
- 에러 발생을 원천 차단
- 블로그: 검색 자동완성, 유효한 카테고리만 표시
- 빈 검색 방지, 필터 조합 시 결과 수 미리 표시

#### 6. 기억보다 인식 (Recognition Rather Than Recall)
- 선택지를 보여주고 기억에 의존하지 않기
- 블로그: 최근 본 글, 관련 글 추천, 검색 히스토리
- 카테고리 목록 항상 표시 (숨기지 말 것)

#### 7. 유연성과 효율성 (Flexibility and Efficiency of Use)
- 초보와 전문가 모두 만족
- 블로그: 키보드 단축키(j/k 다음/이전 글), 검색 필터, RSS
- 고급 기능은 숨기되 접근 가능하게

#### 8. 미적이고 최소한 디자인 (Aesthetic and Minimalist Design)
- 필요한 정보만, 깔끔하게
- 블로그: 콘텐츠 중심, 불필요한 위젯 제거
- 사이드바 간결: 프로필 + 검색 + 카테고리 + (선택적)인기글

#### 9. 에러 인식, 진단, 복구 (Help Users Recognize, Diagnose, and Recover from Errors)
- 에러 메시지가 명확하고 해결책 제시
- 블로그: "검색 결과가 없습니다" + 추천 키워드/인기 글
- 404 페이지에서 홈/검색으로 안내

#### 10. 도움말과 문서 (Help and Documentation)
- 필요 시 쉽게 접근 가능한 도움말
- 블로그: FAQ, 검색 팁, 카테고리 설명
- 이상적으로는 도움말 없이도 사용 가능해야 함

### 블로그 적용 체크리스트
- [ ] 로딩/현재 상태 피드백 (H1)
- [ ] 사용자 언어 사용, 보편적 아이콘 (H2)
- [ ] 뒤로가기/실행취소 정상 작동 (H3)
- [ ] 메뉴/링크/버튼 스타일 일관성 (H4)
- [ ] 빈 검색/잘못된 URL 에러 방지 (H5)
- [ ] 카테고리/최근 글/관련 글 항상 표시 (H6)
- [ ] 검색 필터/키보드 단축키 지원 (H7)
- [ ] 콘텐츠 중심, 최소 UI (H8)
- [ ] 404/빈 결과에 대안 제시 (H9)
- [ ] 검색 팁/카테고리 안내 (H10)

## 3. Aarron Walter "Designing for Emotion"

기능을 넘어 감성까지. Maslow 피라미드의 UX 버전.

### 사용자 니즈 피라미드

```
        △ 즐거움 (Pleasurable)
       ━━━
      사용성 (Usable)
     ━━━━━━━
    신뢰성 (Reliable)
   ━━━━━━━━━━
  기능성 (Functional)
 ━━━━━━━━━━━━━━
```

- **기능성**: 기본 기능 동작 (글 보기, 검색, 카테고리)
- **신뢰성**: 안정적으로 동작 (에러 없음, 빠른 로딩)
- **사용성**: 쉽고 직관적 (Krug + Nielsen)
- **즐거움**: 감성적 연결, 개성, 기억에 남는 경험

### 감성 디자인 원칙

#### 페르소나 (Brand Personality)
- 블로그도 **성격**이 있어야 함
- 기술 블로그: 전문적이면서 친근한 톤
- 일관된 목소리: 글 톤, 마이크로카피, 에러 메시지까지

#### 서프라이즈 & 딜라이트
- 기대 이상의 작은 디테일
- 블로그: 호버 시 부드러운 색상 전환, 코드 복사 버튼, 읽기 진행률 바
- 404 페이지 유머, 이스터에그

#### 심미적 사용성 효과 (Aesthetic-Usability Effect)
- 아름다운 디자인은 **더 사용하기 쉽다고 느낀다** (Norman, 2004)
- 작은 사용성 문제를 시각적 매력이 보상
- 따라서 시각적 품질이 사용성에 직접 영향

#### 신뢰 구축
- 전문적 타이포그래피 → 신뢰
- 일관된 디자인 → 신뢰
- 적절한 여백 → 신뢰
- 깨진 레이아웃/오타 → 신뢰 파괴

### 블로그 적용 체크리스트
- [ ] 기능/신뢰/사용성 기반 확보 (아래서 위로)
- [ ] 블로그 페르소나 정의 (전문적+친근? 진지+깊이?)
- [ ] 마이크로인터랙션: 호버, 전환, 스크롤 애니메이션 (적절히)
- [ ] 서프라이즈 요소: 코드 복사, 읽기 진행률, 목차 자동 하이라이트
- [ ] 일관된 시각 품질 → 신뢰감
- [ ] 404/빈 상태에 개성 있는 메시지
