# Design Review 대가 기준 상세

## 1. Refactoring UI (Adam Wathan & Steve Schoger)

실전 UI 디자인 원칙. 개발자가 디자이너 없이도 좋은 UI를 만들 수 있게 하는 체계적 접근.

### 핵심 원칙

#### 시각적 계층 (Visual Hierarchy)
- **크기만으로 계층을 만들지 말 것** — font-weight, 색상, 간격을 조합
- 1차 정보: 크고 굵게, 높은 대비
- 2차 정보: 보통 크기, 중간 대비 (회색 톤)
- 3차 정보: 작고 가볍게, 낮은 대비
- **레이블은 최후의 수단** — 데이터 자체가 무엇인지 알려줄 수 있으면 레이블 제거

#### 간격과 크기 (Spacing & Sizing)
- **제한된 선택지 시스템 사용** — 4, 8, 12, 16, 24, 32, 48, 64, 96px
- 간격 시스템 없이 매번 "적당히"는 금지
- **요소 간 관계를 간격으로 표현** — 관련 요소는 가깝게, 무관한 요소는 멀리
- 여백으로 그룹핑 (선/구분자 대신 간격)

#### 색상 (Color)
- **HSL 기반 팔레트 구성**: 각 색상 8–10단계 (50~950)
- 회색 팔레트 필수 — 최소 8단계
- **주요색 1개 + 보조색 1–2개**로 제한
- 색상에 의미 부여: 성공(녹), 경고(노), 오류(적), 정보(청)
- **밝은 배경에 밝은 색 텍스트 금지** — 배경색 위 텍스트는 같은 색의 어두운 톤

#### 타이포그래피 (Typography)
- **폰트 크기도 제한된 스케일** — 12, 14, 16, 18, 20, 24, 30, 36, 48px
- 시스템 폰트 스택 우선: -apple-system, system-ui, sans-serif
- 한국어: Pretendard, SUIT, Noto Sans KR 추천
- **em 단위 보다 rem 사용** — 일관성
- line-height: 본문 1.5+, 제목 1.1–1.25

#### 깊이 (Depth)
- 그림자로 깊이 표현 — 5단계 elevation (xs, sm, md, lg, xl)
- **그림자 색상은 회색이 아닌 약간 채도 있는 색** — 더 자연스럽게
- 다크모드에서는 그림자 대신 배경색 밝기 차이로 elevation

### 블로그 적용 체크리스트
- [ ] 제목-본문-메타 간 명확한 시각적 계층
- [ ] 8px 기반 간격 시스템
- [ ] 회색 팔레트 8단계 이상
- [ ] 주요색 + 링크색 + 코드 하이라이트색 정의
- [ ] 시스템 폰트 스택 또는 한국어 웹폰트

## 2. Material Design 3 (Google)

### 핵심 시스템

#### 색상 시스템 (Dynamic Color)
- **5가지 키 색상**: Primary, Secondary, Tertiary, Error, Neutral
- 각 키 색상에서 Tone (0–100) 파생
- **Surface 색상**: 콘텐츠 배경용, Neutral 기반
- **On-Surface**: Surface 위의 텍스트/아이콘 색
- 라이트/다크 테마 모두 자동 대응

#### 타이포그래피 (Type Scale)
- 5가지 역할: Display, Headline, Title, Body, Label
- 각 역할에 Large/Medium/Small 변형
- **Display**: 영웅 텍스트, 가장 큰 크기
- **Headline**: 섹션 제목
- **Title**: 카드/컴포넌트 제목
- **Body**: 본문 (Large: 16px, Medium: 14px)
- **Label**: 버튼, 탭, 캡션

#### Elevation 시스템
- Level 0–5 (0: 표면, 5: 최상단)
- 라이트: box-shadow로 표현
- **다크: 배경 밝기로 표현** (shadow는 안 보이므로)
  - Level 0: surface color
  - Level 1: +5% primary overlay
  - Level 2: +8% primary overlay
  - Level 3: +11% primary overlay

#### 간격 및 그리드
- **4dp 기반 그리드** (dp = density-independent pixel)
- 아이콘: 24dp 기본
- 터치 타겟: 최소 48dp × 48dp
- 콘텐츠 마진: 16dp (모바일), 24dp (태블릿), 적응형 (데스크톱)

### 블로그 적용 체크리스트
- [ ] Primary/Secondary/Surface/On-Surface 색상 정의
- [ ] 다크모드 Surface elevation 적용
- [ ] Type Scale 역할 매핑 (제목→Headline, 본문→Body, 메타→Label)
- [ ] 4px 그리드 준수
- [ ] 터치 타겟 48px 확보 (모바일)

## 3. Nielsen Norman Group (NNG)

### UI 디자인 핵심 원칙

#### F-패턴 레이아웃
- 사용자는 F자 형태로 스캔 — 상단 가로 → 좌측 세로
- **중요 정보는 좌상단**, 덜 중요한 건 우하단
- 블로그: 제목(상단) → 사이드바(좌) → 본문(중앙–우) 패턴 활용

#### 시각적 계층 10원칙 (NNG 정리)
1. **크기**: 크면 중요
2. **색상**: 대비 높으면 주목
3. **대비**: 배경과 차이
4. **정렬**: 그리드 밖 = 눈에 띔
5. **반복**: 패턴 = 일관성
6. **근접성**: 가까운 것 = 관련 있음
7. **여백**: 여백 많으면 중요해 보임
8. **질감/스타일**: 다른 스타일 = 주목
9. **타이포**: 굵기/이탤릭으로 강조
10. **움직임**: 움직이면 주목 (단, 남용 금지)

#### 웹 가독성 가이드라인
- **줄 길이**: 영문 50–75자 최적 (NNG 연구 기반)
- **줄 간격**: 최소 1.5배
- **좌측 정렬**: 양쪽 정렬(justify) 금지 — 불규칙한 단어 간격 발생
- **충분한 여백**: 텍스트 블록 주변 빈 공간
- **청킹(Chunking)**: 긴 텍스트를 소제목/목록으로 분리
- **스캔 가능성**: 볼드, 목록, 소제목으로 핵심 부각

#### 접근성 (Accessibility)
- WCAG 2.1 AA 준수 필수
- 색상 대비 4.5:1 (일반), 3:1 (대형)
- 키보드 탐색 가능
- 포커스 표시 명확
- alt 텍스트 제공
- 색상만으로 정보 전달 금지

### 블로그 적용 체크리스트
- [ ] F-패턴에 맞는 레이아웃
- [ ] 제목/본문/사이드바 명확한 시각적 계층
- [ ] 줄 길이 60–80자
- [ ] 좌측 정렬 (justify 아님)
- [ ] 청킹: 긴 글에 소제목, 목록, 코드블록으로 분리
- [ ] WCAG AA 색상 대비
- [ ] 키보드 탐색 및 포커스 표시

## 레퍼런스 스타일: 박영민 Oracle DBA 블로그 (Tistory)

### 사이드바 레이아웃 핵심 요소

```
┌──────────────────────────────────────────┐
│  [프로필 이미지]                           │
│  블로그 이름                              │
│  간단한 소개                              │
├──────────────────────────────────────────┤
│  🔍 검색                                 │
├──────────────────────────────────────────┤
│  📁 카테고리                              │
│  ├─ Oracle                              │
│  │  ├─ Architecture  (12)               │
│  │  ├─ Performance   (8)                │
│  │  └─ RAC           (5)                │
│  ├─ Linux                               │
│  │  ├─ Shell Script  (3)                │
│  │  └─ 시스템 관리    (7)                │
│  └─ 일상             (4)                │
├──────────────────────────────────────────┤
│  최근 글 / 인기 글                        │
└──────────────────────────────────────────┘
```

### 특징
- **사이드바 고정**: 스크롤해도 항상 접근 가능 (sticky)
- **카테고리 트리**: 접기/펼치기, 게시글 수 표시
- **검색 상단 배치**: 사이드바에서 즉시 접근
- **프로필 간결**: 이미지 + 이름 + 한 줄 소개
- **콘텐츠 중심**: 사이드바는 탐색 도구일 뿐, 주인공은 본문

### 적용 시 주의
- 사이드바 너비: 260–280px 권장
- 모바일: 사이드바 → 햄버거 메뉴 또는 하단 이동
- 카테고리 깊이: 최대 2단계 (3단계부터 복잡)
- 현재 카테고리 하이라이트 필수
