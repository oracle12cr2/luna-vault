### ORA-27366 에러 해결 (PROD3)
- **증상**: `ORA-27366: job ""."" is not running` — CJQ가 반복적으로 유령 job 복구 시도
- **원인**: `SYSMAN.EM_SLM_CALENDAR_UPDATE` job이 인스턴스3에서 유령 RUNNING(status=65) 상태로 잔존
  - `dba_scheduler_running_jobs`에는 없는데 `sys.scheduler$_job` 내부 테이블에 running_instance=3 남아있었음
  - `DBMS_SCHEDULER.STOP_JOB(force=>TRUE)`도 안 먹힘 (이미 안 돌고 있으니까)
- **해결**: `sys.scheduler$_job` 직접 UPDATE → `running_instance=NULL, job_status=1`
  - `UPDATE sys.scheduler$_job SET running_instance=NULL, running_slave=NULL, job_status=1 WHERE obj#=343633;`
- 19:20 이후 에러 재발 없음 확인
- **교훈**: ORA-27366 `"".""`는 scheduler 내부 테이블의 유령 running 엔트리 → STOP_JOB 안 되면 sys 테이블 직접 정리
