### 핵심 스크립트 4개

2. **daytrading.py** (481줄) — 단기 매매: 멀티팩터 스코어링, 09:30 매수 → 14:50 청산
3. **multifactor.py** (187줄) — 스코어링: 기술적(40%) + 수급(40%) + 모멘텀(20%)

### 현재 매매 흐름
- **etf_signal.py**: MA(3/15) 크로스 → 자동 매수/매도 (이미 자동)

  - 14:50 전량 청산 (현재 **승인 모드**)

## 변경 계획 (5단계)

### 1단계: daytrading.py 자동 전환 (즉시 가능)

**목표**: 승인 모드 → 자동 모드로 cron 변경

**변경 내용**:
```bash
# Before (승인 필요)
30 9  * * 1-5  daytrading.py --mode buy
50 14 * * 1-5  daytrading.py --mode sell

# After (자동 실행)
30 9  * * 1-5  daytrading.py --mode buy --auto
50 14 * * 1-5  daytrading.py --mode sell --auto
```

**리스크**: 
- stoploss(-3%)가 이미 10분마다 돌고 있어서 큰 손실 방어 가능
- daytrading은 당일 매수 → 당일 청산이라 오버나잇 리스크 없음

**판단**: ✅ 바로 적용 가능

### 3단계: 유나 AI 매매 실행 권한 (신중하게)

**목표**: 분석 후 직접 daytrading.py 실행

**조건부 자동 실행 규칙**:

| 조건 | 매수 | 매도 |
|------|------|------|
| 멀티팩터 스코어 80+ & 공포지표 NORMAL | ✅ 자동 매수 | - |
| 멀티팩터 스코어 70-80 & 외국인 순매수 | ✅ 자동 매수 | - |
| 멀티팩터 스코어 70-80 & 외국인 순매도 | ❌ 보류 | - |
| 일 손실 -3% 이상 | - | ✅ 즉시 전량 매도 |
| 공포지표 EXTREME (VIX>30) | ❌ 매수 금지 | ⚠️ 보유분 검토 |
| 보유 종목 수익 +10% | - | ✅ 자동 익절 |
| 종목별 비중 40% 초과 | ❌ 추가 매수 금지 | - |

**구현 방법**:
- 기존 daytrading.py의 `--auto` 플래그 사용
- 유나 cron에서 조건 판단 후 SSH로 실행:
  ```bash
  sudo bash -c 'cd /root/.openclaw/workspace/etf-backtest && \
    /home/anaconda3/bin/python3 daytrading.py --mode buy --auto'
  ```

**안전장치**:
- 일일 최대 매매 횟수 제한 (매수 1회, 매도 1회)
- 매매 실행 시 반드시 디스코드 알림
- 판단 근거를 memory/YYYY-MM-DD.md에 기록
- 주간 리뷰 (매주 금요일 분석 리포트)

### 4단계: etf_signal.py 통합 (중기 매매도 유나 판단)

**목표**: 현재 자동인 etf_signal.py에 유나의 추가 판단 레이어 적용

**현재 문제**:
- etf_signal.py는 이미 자동이지만, MA 크로스만으로 판단
- 시장 상황(매크로)을 더 종합적으로 볼 필요 있음

**추가할 판단**:
1. **매크로 분석**: 미국 시장(S&P500, 나스닥) 전일 종가 확인
2. **섹터 로테이션**: 반도체/2차전지/금융 섹터별 강약
3. **변동성 분석**: 최근 5일 변동폭 vs 20일 평균
4. **상관관계**: 종목 간 상관계수 → 분산 효과 확인

**구현**: etf_signal.py의 결과를 유나가 리뷰하고, 필요 시 오버라이드
- signal.py가 매수 신호 → 유나가 최종 검증 후 실행/보류
- signal.py가 매도 신호 → 즉시 실행 (매도는 빠를수록 좋음)
