# 2026-03-31 (월)

## 유나 OpenClaw 설정 문제 완전 해결

- **1차 문제**: 유나 맥북에서 `HTTP 401 authentication_error: Invalid bearer token` 발생
- **2차 문제**: `Unknown model: claude-cli/claude-sonnet-4-6 (model_not_found)` 발생

### 해결 과정
1. **Claude CLI 재인증**: `claude auth login` → OAuth 브라우저 로그인
2. **OpenClaw 설정 수정**: `openclaw configure --section model` 
   - "Anthropic Claude CLI" 선택 (이전에 API key 모드로 잘못 설정됨)
   - `claude-cli/claude-sonnet-4-6`, `claude-cli/claude-opus-4-6` 모델 활성화
3. **게이트웨이 재시작**: `openclaw gateway restart`

### 최종 결과 (정상 작동 확인)
- ✅ Gateway: 정상 연결 (32ms)
- ✅ Discord: ON, OK 상태 
- ✅ 세션: claude-sonnet-4-6 기본 모델로 정상 작동
- ✅ 모든 cron job 정상 실행 중

### 핵심 교훈
- OpenClaw에서 Claude CLI 연동 시 **반드시 "Anthropic Claude CLI" 옵션 선택**해야 함
- API key 모드와 Claude CLI 모드는 완전히 다른 설정
- `claude-cli/` 접두사 모델이 정상 작동하려면 올바른 백엔드 설정 필수
