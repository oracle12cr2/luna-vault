## OEM 13c Cloud Control 설치 진행

### 문제 해결 과정
- ConfigureGC.sh 실행 시 여러 에러 발생:
  1. **"Invalid option sourceLoc" + "Invalid INSTALL type"** → response 파일의 EM_INSTALL_TYPE을 "NOSEED"로 변경하여 해결
  2. **"doesn't contain valid Management Repository"** → `SYSMAN.MGMT_UPGRADEUTIL_TASK`에서 phase=8 레코드 삭제 + CORE status를 1로 변경
  3. **"oracle home already in use"** → `emInstanceMapping.properties` 파일 삭제 (핵심 발견: omsca가 이 파일로 home 사용 여부 판단)
  4. **"OPSS schema not found"** → SYSMAN122140_* 스키마가 없어서 RCU로 재생성 필요

### 핵심 교훈
- `emInstanceMapping.properties` (`$ORACLE_HOME/sysman/config/`) 가 있으면 omsca가 "home already in use"로 판단
- `domain-registry.xml`, `user_projects/` 디렉토리도 정리 필요
- `MGMT_UPGRADEUTIL_TASK` 테이블의 phase=8은 SchemaManager UPGRADE 기록 → ConfigureGC.sh가 "incomplete upgrade"로 판단
- response 파일에서 `EM_INSTALL_TYPE="NOSEED"`, `INSTALL_TYPE="NOSEED"`로 설정해야 함
- SYSMAN122140_* prefix 스키마는 RCU로 별도 생성 필요 (OPSS, IAU, MDS, STB, WLS 등)
