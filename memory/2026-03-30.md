# 2026-03-30 (월)

## 유나 OpenClaw 폴백 문제 디버깅

- **문제**: 유나 맥북에서 Anthropic API rate limit 걸리면 두 번째 프로필로 폴백이 안 됨
- **설정 상태**: `auth.order`에 `anthropic: [luna, yuna]` 순서로 폴백 설정됨
- **로그 분석 결과**:
  - 429 rate_limit → 4번 재시도 후 `failover_decision: rotate_profile` 발동 (폴백 로직 자체는 동작)
  - 프로필 전환 후에도 실패 — 두 번째 프로필 토큰이 실제로 저장되지 않았기 때문
- **근본 원인**: 설정 파일에 프로필 이름(`anthropic:luna`, `anthropic:yuna`)만 있고, 실제 API 토큰/OAuth 크레덴셜이 macOS 키체인에 저장되지 않음. `mode: "token"`으로 되어있는데 키가 없음.
- **다음 할 일**: 유나 맥북에서 두 계정 모두 `openclaw auth` 또는 `openclaw configure`로 토큰 재등록 필요. Claude Code OAuth 토큰이면 `mode: "oauth"`로 변경 필요.
- **해결 완료!** 유나 맥북에서 `openclaw configure --section model` → "Anthropic Claude CLI" 선택 → OAuth 브라우저 로그인으로 크레덴셜 등록 성공
- API 키 방식은 Console 크레딧 충전 필요해서 실패 → OAuth가 정답
- 교훈: OpenClaw auth 프로필의 `mode: "token"`은 내부적으로 OAuth. 실제 크레덴셜은 `~/.openclaw/credentials/oauth.json`에 저장돼야 함. `openclaw configure`로 브라우저 OAuth 플로우를 한 번은 거쳐야 동작함
