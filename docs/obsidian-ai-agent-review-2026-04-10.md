# Obsidian AI Agent 저장소 검토 메모 (2026-04-10)

대상 저장소:
- `https://github.com/groundroot/obsidian-ai-agent`

## 한 줄 요약

Obsidian 볼트를 AI가 이해하고 연결해주는 플러그인 계열이며, 특히 **Ollama 기반 로컬 LLM 지원**을 강화한 포크 버전으로 보인다. OpenClaw의 대체재라기보다, **Obsidian 안에서 지식 탐색/정리/초안 작성**을 돕는 보완 도구에 가깝다.

---

## 핵심 개념

이 프로젝트는 단순 챗봇이 아니라 다음 기능들을 지향한다.

- 노트 임베딩 생성
- 의미 기반 유사 노트 검색
- 노트 간 관계 분석
- 지식 갭 탐색
- 현재 노트/새 노트 초안 작성

즉 성격상 **세컨드 브레인 보조 도구**에 가깝다.

---

## 눈에 띈 특징

### 1. 로컬 LLM(Ollama) 강화
가장 큰 차별점은 이 부분이다.

- Ollama를 통한 로컬 생성 모델 사용
- Ollama를 통한 로컬 임베딩 모델 사용
- OpenAI 호환 API + native API fallback 지원
- Gemma, Qwen, Llama 계열 사용 흐름 강조

남궁건님의 로컬/자체 인프라 선호와 잘 맞을 가능성이 있다.

### 2. Mac/Apple Silicon 친화 흐름
README에서 Mac 사용자와 MLX 관련 흐름을 별도로 언급한다.

- Apple Silicon 환경 고려
- Gemma/MLX 추천 흐름 포함

### 3. 상태 추적 최적화
노트 상태를 frontmatter 기반으로 추적한다고 설명한다.

예:
- `osba.indexedAt`
- `osba.embeddingHash`
- `osba.analyzed`
- `osba.analysisHash`

즉 이미 최신 상태인 노트는 다시 인덱싱/재분석하지 않도록 설계한 것으로 보인다.

- 로컬 Ollama 조합 시 비용 절감 및 프라이버시 측면 이점

---

## 우리 환경 기준 한계

### 1. OpenClaw 대체재는 아님
이 프로젝트는 행동하는 자동화 엔진이 아니다.

OpenClaw와 비교하면:
- OpenClaw: 툴 실행, 세션 orchestration, cron, ACP, SSH, 실작업 자동화
- obsidian-ai-agent: Obsidian 내부의 검색/연결/분석/초안 보조

즉 관계는 **경쟁재가 아니라 보완재**에 가깝다.

### 2. README는 설명형, 마케팅형 성격이 강함
장점 소개와 비유는 많지만, 실제 내부 구조/한계/성능/운영상 제약 설명은 더 확인이 필요해 보인다.

### 3. 플러그인 복잡도
지원 범위가 넓다.

- 인덱싱
- 분석
- 초안 작성
- 유사 노트 검색
- 작업 큐
- 예산 관리
- 로컬/클라우드 모델 혼합 지원

이 정도면 실제 사용 시 설정 꼬임, UX 버그, 성능 이슈가 있을 가능성을 염두에 둬야 한다.

### 4. frontmatter 오염 가능성
노트마다 `osba.*` 필드가 많이 추가되면 실제 볼트 운영에서 거슬릴 수 있다.

## 기술적으로 설치 전 확인할 것

실제 도입 전에는 아래를 꼭 확인해야 한다.

1. GitHub Release 자산 존재 여부
   - `main.js`
   - `manifest.json`
   - `styles.css`
   - BRAT 설치 흐름이 실제로 성립하는지

2. 인덱싱/임베딩 성능
   - sql.js 기반 대규모 볼트 처리 성능
   - 재분석/재인덱싱 정책이 실제로 잘 동작하는지

3. 한국어 품질
   - 한국어 노트에서 유사도 검색 품질
   - 관계 분석 정확도
   - 초안 생성 실용성

4. 로컬 모델 조합 품질
   - 생성 모델: Gemma/Qwen/Llama 계열
   - 임베딩 모델: `nomic-embed-text`
   - 실제 체감 속도와 품질 균형

## 권장 도입 방식

메인 볼트에 바로 넣기보다 아래처럼 테스트하는 것이 좋다.

### 1단계: 작은 테스트 볼트
- 노트 30~100개 수준
- 공부/프로젝트 노트 혼합

### 2단계: 로컬 모드 우선
- 생성 모델: `qwen2.5:7b` 또는 `gemma`
- 임베딩 모델: `nomic-embed-text`

### 3단계: 검증 포인트
- 비슷한 노트 검색 품질
- 연결 추천의 납득 가능성
- 초안 생성 실효성
- 인덱싱 속도
- frontmatter 증가 허용 가능성
