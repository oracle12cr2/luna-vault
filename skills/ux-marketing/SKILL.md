# UX/Marketing — 기술 블로그 전문가

## 권위 기준 (Masters)

이 스킬은 3대 권위 기준을 따른다:

1. **Steve Krug "Don't Make Me Think"** — 직관적 사용성의 바이블
2. **Jakob Nielsen 10 Heuristics** — 사용성 평가의 골든 스탠다드
3. **Aarron Walter "Designing for Emotion"** — 기능 너머 감성 디자인

> 상세 기준은 `references/masters.md` 참조.

## 🚧 앞태클 (Pre-check) — 작업 전 필수 확인

UX/마케팅 작업을 시작하기 **전에** 반드시 확인. 확인 안 되면 작업 시작하지 말 것.

1. **타겟 사용자는 누구?** — 주니어 개발자? 시니어 DBA? 비개발자?
2. **이 페이지/컴포넌트의 핵심 목표?** — 글 발견? 구독? 신뢰 구축?
3. **사용자 유입 경로?** — 검색엔진? SNS 공유? 직접 방문?
4. **기존 디자인 시스템/브랜드 가이드 있는지?**
5. **핵심 KPI?** — 페이지뷰? 체류시간? 이탈률? 구독?

확인 방법:
- 사용자에게 직접 질문
- 기존 사이트 구조/코드 분석
- 스크린샷/URL이 있으면 직접 분석

## 📐 수치 기준 (Quantitative Checks)

AI가 자동으로 검증할 수 있는 수치 기준:

### 네비게이션 & 정보구조
| 항목 | 기준 | 근거 |
|------|------|------|
| 클릭 깊이 | 모든 콘텐츠 3클릭 이내 도달 | Krug 3-click rule |
| 카테고리 깊이 | 최대 2단계 중첩 | Nielsen: 깊은 계층 = 미로 |
| 메뉴 항목 수 | 상단 메뉴 7±2개 이내 | Miller's Law |
| 검색 접근성 | 모든 페이지에서 1클릭 내 검색 가능 | Krug 필수 |
| 브레드크럼 | 2단계 이상 깊이에서 필수 표시 | Nielsen |
| 현재 위치 표시 | 네비게이션에서 현재 페이지 하이라이트 | Krug |

### 가독성 & 콘텐츠
| 항목 | 기준 |
|------|------|
| F-패턴/Z-패턴 | 주요 정보가 F/Z 스캔 경로에 배치 |
| 스캔 가능성 | 소제목 3–5 스크롤마다 1개, 목록/코드블록 활용 |
| 단락 길이 | 최대 4–5줄 (모바일 기준) |
| CTA 위치 | 글 하단 + 사이드바에 명확한 CTA |
| 글 메타 정보 | 작성일, 카테고리, 예상 읽기 시간 표시 |

### 성능
| 항목 | 기준 | 검증 방법 |
|------|------|-----------|
| 페이지 로드 | 3초 이내 (FCP) | Lighthouse/PageSpeed |
| LCP | 2.5초 이내 | Core Web Vitals |
| CLS | 0.1 이하 | Core Web Vitals |
| INP | 200ms 이하 | Core Web Vitals |
| 이미지 최적화 | WebP/AVIF, lazy loading | 코드 확인 |

### 모바일 UX
| 항목 | 기준 |
|------|------|
| 터치 타겟 | 최소 44×44px (Apple HIG) / 48×48dp (Material) |
| 터치 타겟 간격 | 최소 8px |
| 텍스트 크기 | 모바일 본문 최소 16px (iOS 줌 방지) |
| 뷰포트 메타 | `<meta name="viewport" content="width=device-width, initial-scale=1">` |
| 가로 스크롤 | 절대 금지 (콘텐츠 영역) |

## 👁️ 사람 눈 기준 (Visual Scenarios)

수치로 잡을 수 없는 항목. 시나리오 기반으로 판단:

### 첫인상 5초 테스트
- 페이지를 5초만 봤을 때:
  - 이 블로그가 **무엇에 관한** 블로그인지 알 수 있는가?
  - **어디를 클릭해야** 원하는 글을 찾을 수 있는지 명확한가?
  - **신뢰감**이 느껴지는가? (전문적 인상)
- "뒤로 가기"를 누르고 싶은 요소가 있는가?

### 네비게이션 직관성
- 처음 방문한 사용자가 **생각 없이** 원하는 곳에 갈 수 있는가?
- 메뉴 명칭이 사용자 언어인가? (개발자 용어 vs 일반 용어)
- 현재 위치를 항상 알 수 있는가?
- 뒤로 돌아가기가 쉬운가?

### 사용자 여정 시나리오
1. **검색 유입**: 구글 → 글 → 관련 글 발견 → 카테고리 탐색 → 구독
2. **직접 방문**: 홈 → 최신 글 확인 → 관심 카테고리 → 깊이 읽기
3. **SNS 유입**: 공유 링크 → 글 → 저자 프로필 → 다른 글 → 북마크

각 여정에서 **막히는 지점**이 없는지 체크.

### 감성 디자인 (Walter)
- 기능적 요구 충족 후, **즐거움/개성**이 느껴지는가?
- 브랜드 개성이 일관되게 표현되는가?
- 미세한 디테일 (호버 효과, 전환, 마이크로카피)에 정성이 느껴지는가?

## 🚫 안티패턴 (절대 하면 안 되는 것)

1. **사이드바 과밀** — 위젯/배너/광고로 사이드바가 콘텐츠보다 길어짐
2. **카테고리 미로** — 3단계 이상 중첩, 비슷한 이름의 카테고리 난립
3. **팝업 남발** — 구독/쿠키/알림 팝업이 글 읽기를 방해
4. **검색 숨김** — 검색창이 메뉴 깊숙이 숨어있거나 아예 없음
5. **무한 스크롤 남용** — 페이지네이션 없이 끝없는 스크롤 (블로그에 부적합)
6. **자동 재생** — 영상/음악 자동 재생
7. **모바일 무시** — 데스크톱만 고려한 레이아웃
8. **빈 상태 무처리** — 검색 결과 0건, 카테고리 0건일 때 안내 없음
9. **일관성 없는 네비게이션** — 페이지마다 메뉴 구조 다름
10. **CTA 과잉/부재** — CTA가 너무 많아 피로하거나, 아예 없어 행동 유도 실패
11. **뒤로가기 함정** — SPA에서 뒤로가기가 이상하게 작동
12. **느린 로딩 무피드백** — 로딩 중 아무 표시 없음 (스켈레톤/스피너 필요)

## ✅ 백태클 (Post-validation) — 작업 후 자체 검증

작업 완료 후 반드시 아래 루프를 실행:

### Step 1: 수치 기준 자동 검증
```
□ 모든 콘텐츠 3클릭 이내 도달 가능
□ 카테고리 깊이 ≤ 2단계
□ 상단 메뉴 ≤ 9개
□ 모든 페이지에서 검색 1클릭 접근
□ 브레드크럼 2단계+ 에서 표시
□ 현재 위치 하이라이트
□ F/Z 패턴 준수
□ CTA 배치 확인 (글 하단 + 사이드바)
□ 글 메타 정보 (날짜, 카테고리, 읽기 시간)
□ 터치 타겟 ≥ 44px
□ 모바일 본문 ≥ 16px
□ 뷰포트 메타 태그
□ 가로 스크롤 없음
```

### Step 2: 미달 시 자동 수정
- 미달 항목 발견 → **사용자에게 보고하지 말고** 즉시 수정
- 수정 후 Step 1 재실행
- 모든 항목 통과할 때까지 반복

### Step 3: 사람 눈 기준 보고
모든 수치 기준 통과 후에만 아래 시나리오 결과를 사용자에게 보고:
- 첫인상 5초 테스트 결과
- 네비게이션 직관성 평가
- 3가지 사용자 여정 시나리오 평가
- 감성 디자인 평가
- 안티패턴 위반 여부

### 보고 형식
```
## UX/마케팅 리뷰 결과

### 수치 기준 (전체 통과 ✅)
- 클릭 깊이: ✅ (최대 2클릭)
- 메뉴 구조: ✅ (상단 5개, 카테고리 2단계)
- 모바일: ✅ (터치 48px, 본문 16px)
- 성능: ✅ (FCP 1.8s, LCP 2.1s)
- ...

### 사용자 경험 평가
- 첫인상: [평가]
- 네비게이션: [평가]
- 사용자 여정: [평가]
- 감성: [평가]

### 개선 제안 (있을 경우)
1. ...
```
