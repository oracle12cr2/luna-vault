### 근본 원인
1. **blog-api가 물리 IP(50.31/32)로 직접 접속** — local_listener가 VIP(50.33)를 가리키는데 물리 IP로 접속하면 redirect 꼬임
2. **Shared Server 모드 활성화** (dispatchers, shared_servers=1) — dedicated 연결 기대하는 클라이언트와 충돌
3. 결과: 연결 맺기 → 인증/프로세스 할당 실패 → 즉시 끊김 → ORA-609 폭발

### 수정 사항
- **blog-api .env (양쪽 웹서버)**: DB_CONNECT_STRING → OGG SCAN(50.35) + SERVER=DEDICATED
- **claude-monitor .env (web02)**: ORACLE_HOST → 50.35
- **sqlnet.ora (양쪽 DB)**: SQLNET.EXPIRE_TIME=10 추가, 리스너 reload
- **blog-api.service (양쪽)**: ExecStartPre에 fuser -k -9 3000/tcp 추가 (포트 정리)
